# Follow Up Module (Sequential Automation + Kanban Column Selection + Anti-Enqueueing) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the "Follow Up" module allowing users to configure sequential re-engagement workflows per Kanban column status, with fixed/AI message generation, customizable sending windows, automatic cancellation on reply/status change, and staggered anti-enqueueing scheduling.

**Architecture:** New SQLAlchemy models `FollowupConfig`, `FollowupStep`, and `LeadFollowup` persist rules and execution queues. An asynchronous scheduler service in FastAPI runs every 60 seconds, processing due follow-ups while enforcing sending windows and anti-enqueueing spacing. A new Next.js page `/follow-up` provides full UI configuration and live queue management.

**Tech Stack:** Python 3.11, FastAPI, SQLAlchemy (Async), PostgreSQL, `openai` Python SDK, Next.js 15, Tailwind CSS, Lucide icons, Shadcn UI components.

## Global Constraints
- Only 1 active Follow Up sequence configuration (`FollowupConfig` id=1).
- Target Kanban columns filterable by user (`target_statuses`, default `["novo", "em_atendimento"]`). Leads in `ganho` or `perdido` do not receive follow-ups.
- Staggering spacing: Minimum `min_interval_minutes` (default 4 mins) + random jitter between tasks.
- Navigation: Sidebar item "Follow Up" pointing to `/follow-up`.

class FollowupConfig(Base):
    __tablename__ = "followup_configs"

    id = Column(Integer, primary_key=True, default=1)
    is_enabled = Column(Boolean, default=False)
    target_statuses = Column(JSON, default=lambda: ["novo", "em_atendimento"])
    window_start = Column(String(5), default="08:00")
    window_end = Column(String(5), default="20:00")
    min_interval_minutes = Column(Integer, default=4)
    updated_at = Column(DateTime(timezone=True), onupdate=func.now(), server_default=func.now())

class FollowupStep(Base):
    __tablename__ = "followup_steps"

    id = Column(Integer, primary_key=True, index=True)
    step_number = Column(Integer, nullable=False)
    delay_hours = Column(Integer, nullable=False)
    mode = Column(String(20), default="text")
    content = Column(Text, default="")
    created_at = Column(DateTime(timezone=True), server_default=func.now())

class LeadFollowup(Base):
    __tablename__ = "lead_followups"

    id = Column(Integer, primary_key=True, index=True)
    lead_id = Column(Integer, ForeignKey("leads.id"), nullable=False)
    step_id = Column(Integer, ForeignKey("followup_steps.id"), nullable=True)
    step_number = Column(Integer, nullable=False)
    scheduled_at = Column(DateTime(timezone=True), nullable=False)
    status = Column(String(20), default="scheduled") # scheduled, sent, cancelled, failed
    sent_at = Column(DateTime(timezone=True), nullable=True)
    cancel_reason = Column(String(255), nullable=True)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
```

from datetime import datetime, timezone, timedelta
from app.services.followup_calculator import adjust_for_sending_window, stagger_schedule_time

def test_adjust_for_sending_window():
    # 03:00 AM target -> should adjust to 08:00 AM of the same day
    dt_night = datetime(2026, 7, 21, 3, 0, tzinfo=timezone.utc)
    adjusted = adjust_for_sending_window(dt_night, "08:00", "20:00")
    assert adjusted.hour == 8
    assert adjusted.minute == 0

def test_stagger_schedule_time():
    # 2 tasks targeted at the exact same minute -> second task should be pushed forward by at least min_interval
    t1 = datetime(2026, 7, 21, 10, 0, tzinfo=timezone.utc)
    existing = [t1]
    t2_raw = datetime(2026, 7, 21, 10, 1, tzinfo=timezone.utc)
    staggered = stagger_schedule_time(t2_raw, existing, min_interval_minutes=4)
    assert staggered >= t1 + timedelta(minutes=4)
```

### Task 4: Background Follow Up Execution Worker

**Files:**
- Create: `backend/app/services/followup_scheduler.py`
- Modify: `backend/app/main.py`

- [ ] **Step 1: Implement background worker service**

In `backend/app/services/followup_scheduler.py`:
Async function `run_followup_worker_loop()` running inside FastAPI `lifespan`:
- Polls `lead_followups` table every 60s for items with `status == "scheduled"` and `scheduled_at <= NOW()`.
- Validates that lead status is in `target_statuses` and lead hasn't sent a message recently.
- If mode is `"text"`: replaces `{nome}` and sends via WAHA (`POST /api/sendText`).
- If mode is `"ai"`: calls OpenAI to generate re-engagement text using lead conversation context and sends via WAHA.
- Updates status to `"sent"` or `"failed"`.
